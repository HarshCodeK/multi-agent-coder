# Multi-Agent Coder

A tool that takes a plain-English app description and uses 3 AI agents working in sequence to generate a complete, working project. Type what you want, and the agents plan the architecture, break it into file-level tasks, and write every line of code.

## Why 3 separate agents?

Planning, architecture, and implementation are different kinds of thinking. A single model asked to do everything at once tends to produce vague or inconsistent results. By splitting the work into three focused stages — each with its own prompt and JSON contract — we get more reliable, structured output. Each agent builds on the previous one's work without distraction.

```
Prompt (your idea)
    |
    v
Planner Agent  -->  Project plan (tech stack, features, files)
    |
    v
Architect Agent -->  Tasks per file (filename + description)
    |
    v
Coder Agent     -->  Code written to disk
    |
    v
Generated Project Files
```

## How to run

1. Clone the repo and install dependencies:
   ```
   pip install -r requirements.txt
   ```

2. Copy `.env.example` to `.env` and add your Groq API key:
   ```
   GROQ_API_KEY=gsk_your_key_here
   ```

3. Start the UI:
   ```
   streamlit run app.py
   ```

4. Type an app description (e.g. "Build a calculator web app with add, subtract, multiply and divide buttons") and click **Generate Project**.

## Worked example

**Prompt:** "Build a calculator web app with add, subtract, multiply and divide buttons"

**Plan generated by the Planner Agent:**
```json
{
  "project_name": "calculator_web_app",
  "tech_stack": ["html", "css", "javascript"],
  "features": ["add", "subtract", "multiply", "divide", "error_handling"],
  "file_list": ["index.html", "style.css", "script.js", "README.md"]
}
```

**Files generated:**
- `index.html` — the calculator UI with display + number/operator buttons
- `styles.css` — layout, colors, button styling
- `script.js` — calculation logic, input handling, error handling
- `README.md` — project overview

All files are written to `generated_projects/calculator_web_app/` and can be downloaded as a ZIP.

## What I'd add next

- **Reviewer Agent** — checks generated code for errors and loops back to the Coder to fix them automatically
- **Edit/Regenerate** — allow the user to modify or regenerate individual files without re-running the full pipeline
- **Template library** — pre-built prompts for common project types (API, dashboard, CLI tool) to get faster, more consistent results
